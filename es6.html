When to use these ES6 features

Spread & Rest

  • Copy API arrays before mutating; avoid side effects.
  • Merge new items into API results without touching originals.
  • Capture “everything else” without listing every field.

Destructuring

  • Pluck nested fields with defaults to handle missing data.
  • Rename fields to match your UI layer.
  • Make intent obvious when shaping objects.

Map / Filter / Reduce

  • Map: normalize and format rows for display.
  • Filter: keep only users that pass a rule (e.g., active).
  • Reduce: compute aggregates for dashboards.

Template literals

  • Build readable strings for UI labels and logs.
  • Embed variables inline instead of concatenation.
  • Keep formatting in one place.
